# Oracle

## Các tính năng cơ bản của Oracle Database

Oracle Database cho phép bạn lưu trữ và truy suất dữ liệu một cách nhanh chóng và an toàn. Dưới đây là một số ưu diểm của Oracle Database:
- **Oracle Database có tính nền tảng:** Có thể chạy trên nhiều loại phần cứng và hệ điều hành khác nhau, bao gồm Windows Server, Unix và nhiều bản phân phối GNU/linux
- **Khả năng mạng (Networking):** Oracle db có ngăn xếp mạng cho phép các ứng dụng từ nhiều nền tảng khác nhau giao tiếp liền mạch với Oracle Db. Ví dụ, các ứng dụng chạy trên Windows có thể kết nối đến máy chủ Oracle db đang chạy trên Unix
- **Tuân thủ ACID:** Oracle là một cơ sở dữ liệu tuân thủ ACID, giúp duy trì tính toàn vẹn và độ tin cậy của dữ liệu 
- **Cam kết với các công nghệ mở:** Oracle là một trong những hệ quản trị cơ sở dữ liệu đầu tiên hỗ trợ GNU/Linux từ cuối những năm 1990, trước khi GNU/Linux trở thành một sản phẩm thương mại. Oracle đã liên tục hỗ trợ nền tảng mở này kể từ đó

Oracle Database có nhiều đặc điểm cấu trúc khiến nó trở nên phổ biến:

- **Cấu trúc dữ liệu logic:** Oracle sử dụng cấu trúc dữ liệu logic để lưu trữ dữ liệu, cho phép người dùng làm việc với cơ sở dữ liệu mà không cần biết vị trí lưu trữ vật lý của dữ liệu
- **Phân vùng (Partitioning):** Là một tính năng hiệu năng cao, cho phép bạn chia một bảng lớn thành nhiều phần riêng biệt và lưu trữ mối phần trên các thiết bị lưu trữ khác nhau
- **Bộ nhớ đệm (Memory caching):** Kiến trúc bộ nhớ đệm cho phép mở rộng một cơ sở dữ liệu rất lớn mà vẫn duy trì hiệu năng cao
- **Tử điển dữ liệu (Data Dictionary):** Là tập hợp các bảng và các view nội bộ giúp việc quản trị Oracle DB hiệu quả hơn
- **Sao lưu và khôi phục (Backup and Recovery):** Đảm bảo tính toàn vẹn dữ liệu trong trường hợp xảy ra sự cố hệ thống. Oracle cung cấp công cụ mạnh mẽ mang tên `Recovery Manager (RMAN)`, cho phép các DBA thực hiện sao lưu lạnh (cold), sao lưu nóng (hot), sao lưu gia tăng (incremental) và khôi phục dữ liệu theo thời điểm (point-in-time recovery)
- **Cụm máy chủ (Clustering) - Oracle Real Application Clusters(RAC):** Oracle hỗ trợ tính sẵn sàng cao (high availability), cho phép hệ thống tiếp tục hoạt động mà không bị gián đoạn dịch vụ ngay cả khi một hoặc nhiều máy chủ trong cụm gặp sự cố

## Máy chủ Oracle Database cơ bản
Oracle DB sử dụng kiến trúc client-server. Kiến trúc client-server là mô hình trong đó client gửi yêu cầu dịch vụ và server (Oracle DB) cung cấp các dịch vụ đó

## Kiến trúc Oracle Database
### Client
Phía Client có thể là một ứng dụng hoặc máy tính của người dùng. Đây là công cụ cho phép bạn tương tác với Oracle Database. Một số công cụ client của Oracle bao gồm SQL Developer và SQL*Plus.

### Server
Server là nơi Oracle Database được cài đặt và vận hành. Server bao gồm:
- **Listener:** Một tiến trình nền lắng nghe các yêu cầu kết nối từ client
- **Oracle Instance:** Tập hợp các tiến trình quản lý cơ sở dữ liệu
- **Oracle Database:** Dữ liệu thực tế được lưu trữ trên đĩa, chẳng hạn như các bảng (table)

**NOTE**: client và server có thể được cài đặt trên cùng một máy tính.
